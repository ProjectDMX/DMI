# 2026-01-16 DB DB 集成计划

## 目标
把 host_engine 的 submit 流程整合进 `MonitoringEngine`，做到每个 step 结束后自动完成：  
`cache_dict` 物化 -> tensor 转 CPU -> 写入后端数据库。用户不再手动调用 `put_batch/submit`。

## 现有参考流程（proj-dmx-host-side）
- `run_with_cache` 得到 `cache_dict`
- `BackendFuture.result()` 物化
- 由 `last_hidden_state` 计算 logits
- 全部 tensor 转 CPU
- `put_batch(host_engine, [key], [start_token_idx], [cache_dict], True)`

## 计划组件
1) **DBConfig（新）**
- 字段：`enabled`, `backend`, `mem_mb`
- request key 信息：`model_id`, `request_id`, `request_key_builder`

2) **HostEngine（已有，复用）**
- 由 `init_host_engine(...)` 初始化
- 新 API：`host_engine.submit(keys, start_token_idx, cache_dicts)`
- host_engine 内部线程负责 `cache_dict` 物化 + CPU 组装 + 写库

3) **Engine 集成**
- `MonitoringEngine` 持有 host_engine（config 开启时初始化）
- `MonitoringEngine.close()` 时关闭 host_engine
- 增加 request 上下文接口：  
  - `set_request_context(model_id, request_id, token_offset)`  
  - 或 `begin_request(request_id)` + `update_token_offset(delta)`

4) **Hook/Cache 集成**
- 在 `HookedRootModule.run_with_cache` 中注册 step 级 cache：  
  `engine.register_step_cache(step_id, cache_dict, token_ids, phase)`
- `end_step()` 后调用 `host_engine.submit(...)`

## 每步数据流
1. `start_step()` -> forward -> `run_with_cache` 返回 `cache_dict`
2. `register_step_cache(...)` 暂存 cache + token_ids
3. `end_step()` 提交 native backend
4. host_engine.submit(...) 在后台完成：  
   - `future.result()` 物化  
   - `logits` 由 backend offload 生成并放进 `cache_dict`  
   - `cache_dict["token_ids"] = token_ids`  
   - 所有 tensor 转 CPU  
   - 写入数据库（host_engine 内部）

## 同步/异步说明（现在由 host_engine 内部处理）
- `MonitoringEngine` 只负责在 `end_step()` 后调用 `host_engine.submit(...)`  
- 同步/异步策略由 host_engine 内部线程决定  
- 重点是确保 cache_dict 在清理前被 host_engine 消费

## 推荐链路（native 异步拷贝 + host_engine 异步写库）
1) `end_step()` 提交 native backend  
2) native backend 异步 GPU→CPU 拷贝  
3) `host_engine.submit(...)` 入队  
4) host_engine 线程调用 `future.result()`（阻塞等待 ready）  
5) `future.result()` 在 C++ 侧释放 result slot  
6) host_engine 继续写库  

## 现有代码中的内存移动链路（GPU→DB）
**1) GPU 上产生 hook tensor**  
- hook 在 forward 中触发，原始 tensor 在 GPU  
- 见 `HF_Prometheus/transformers/src/transformers/models/gpt2_p/hook_points.py`

**2) native backend 异步搬运**  
- `MON_NATIVE_TO_CPU=1` 时，native backend 做 GPU→CPU 拷贝  
- future ready 后才能被 `result()` 取出  
- 见 `HF_Prometheus/monitoring/csrc/api_submit.cpp::future_result`

**3) Python 侧物化与 CPU 组装**  
- `BackendFuture.result()` 返回 CPU tensor  
- Python 侧补充 `token_ids` / `final_logits`

**4) 写入数据库**  
- `host_engine.submit(...)` 内部把 CPU tensor 写入 DB  
- 见 `proj-dmx-host-side/benchmark/integrated_decode.py`

## 实现计划（含代码段，供 review）
### 1) 新增 DBConfig
文件：`HF_Prometheus/monitoring/config.py`（或新文件 `monitoring/db_config.py`）
```python
@dataclass
class DBConfig:
    enabled: bool = False
    backend: str = "clickhouse-driver"
    mem_mb: int = 2048

    # request key 生成
    request_key_builder: Optional[Callable[[str, str], object]] = None

```

### 2) MonitoringEngine 持有 host_engine
文件：`HF_Prometheus/monitoring/engine.py`
```python
class MonitoringEngine:
    def __init__(..., config: Optional[MonitoringConfig] = None, db: Optional[DBConfig] = None):
        self.config = config
        self.db = init_host_engine(...) if db and db.enabled else None
        self._pending_db: dict[int, dict] = {}

    def register_step_cache(self, step_id: int, payload: dict) -> None:
        self._pending_db[step_id] = payload

    def end_step(self) -> None:
        ...
        # 原本提交 native backend 后，触发 db
        payload = self._pending_db.pop(step_id, None)
        if payload and self.db is not None:
            self.db.submit(payload["keys"], payload["start_token_idx"], payload["cache_dicts"])

    def close(self) -> None:
        ...
        if self.db is not None:
            terminate_host_engine(self.db)
```

### 3) run_with_cache 注册 step 级 payload
文件：`HF_Prometheus/transformers/src/transformers/models/gpt2_p/hook_points.py`
```python
engine = self.monitoring_engine
if engine is not None:
    payload = {
        "step_id": int(getattr(engine, "_current_step_id", 0)),
        "cache_dict": cache_dict,
        "token_ids": model_kwargs.get("input_ids", None),
        "keys": [engine.current_request_key()],          # list
        "start_token_idx": [engine.current_token_offset()],
        "cache_dicts": [cache_dict],
    }
    engine.register_step_cache(payload["step_id"], payload)
```

### 4) 早清理风险控制
建议默认：
- `MON_NATIVE_AUTOCLEAR=0`
- 主线程不要调用 `resolve_all()` / `clear_completed_results()`
- host 线程尽快消费 `future.result()`
