DMI Node-Toggle

Enable/disable per-hook producer nodes inside an already-captured CUDA graph, between replays, via cudaGraphNodeSetEnabled — no re-capture. Pay monitoring overhead in proportion to the hooks you actually capture.

Mechanism — 4 steps

1

Capture & record

vLLM captures the FULL decode graph. DMI's producer op records its node via cudaStreamGetCaptureInfo. keep_graph=True so handles survive instantiate.

enable_toggle_capture(True)
2

Bind

After warmup, register each captured graph's exec handle, keyed by its cudaGraph_t.

bind_graph_exec(graph, exec)
3

Toggle

Flip the enabled set at runtime → cudaGraphNodeSetEnabled on the registered nodes. No re-capture, no re-instantiate.

set_active_hooks(enabled)
4

Replay

Enabled nodes fire (D2D→ring→export); disabled nodes are skipped by the replay engine (~0.34 µs/node residual).

graph.replay()

Apply strategies — eager vs lazy

Eagerdefault

set_active_hooks applies the new set to all bound graphs immediately.

  • simple; reconfigure cost ∝ changed × all-graphs
  • pybind-batched effective-set query (one C++ call, not N)

Lazyopt-in

set_active_hooks_lazy bumps a version; each graph is applied on its first replay (hook in CUDAGraph.replay(), uses vLLM's own dispatch).

  • cost ∝ changed × graphs-used (often 1–2 of N)
  • per-exec event guard before mutating an executing exec
