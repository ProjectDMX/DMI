# 存储与索引（MVP 暂缓）

本模块在 MVP 中不纳入范围。当前阶段仅提供：
- GPU→CPU 非阻塞传输与 CPU pinned ring 内存态缓存；
- 实时订阅流（gRPC/WebSocket/IPC）供可视化/调试/对齐使用；
- 轻量“最近 N 秒”窗口的进程内查询。

后续阶段（非 MVP）再设计与实现：
- 落盘格式（分段二进制/列式）与索引结构；
- 轮转与清理策略、崩溃恢复与一致性；
- 性能/容量权衡与压缩方案。

说明：推理透明性 MVP 的目标是最小侵入与实时性保障，落盘与索引延后不影响闭环和验证。

- 当进入后续阶段再新增：段格式、索引与轮转的实现与参数通路。
